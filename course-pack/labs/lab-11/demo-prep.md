## üé¨ Part 1: 90-Second Video Production

### Why This Matters

Your 90-second video will:
- Be embedded in your README (first thing recruiters see)
- Drive signups if you launch publicly
- Show your presentation skills
- Account for 1.5 points of your final grade

**Analysis of 500+ SaaS launches shows:** Talking-head format drives 100+ new signups in 24 hours.

---

### The Formula (Tested on 500+ Launches)

**0-15s: The Hook (Face on Camera)**
- Start with founder's face
- State the pain point emotionally
- Create immediate connection

**15-45s: The Solution (UI Overlay)**
- Show the product working
- Focus on the "magic moment"
- Voiceover while screen recording plays

**45-75s: The Value (Mix)**
- Show metrics, results, testimonials
- Cut between face and data/screenshots
- Prove the impact

**75-90s: Call to Action (Face on Camera)**
- Back to founder's face
- State the URL clearly
- Simple thank you

## Recording Checklist

### Before Recording

- [ ] Script finalized (all 4 sections written)
- [ ] Quiet room secured
- [ ] Good lighting (face the window or use desk lamp)
- [ ] Phone/laptop fully charged
- [ ] Test recording short clip (check audio/video)
- [ ] Screen recording ready (app deployed and tested)

### Camera Setup (Talking-Head Segments)

- [ ] Camera at eye level
- [ ] Face well-lit (no shadows)
- [ ] Clean/blurred background
- [ ] Phone in airplane mode (no interruptions)
- [ ] Record several takes (pick best one)

### Screen Recording Setup (UI Demo)

- [ ] Close unnecessary tabs/windows
- [ ] Hide personal info (email, API keys)
- [ ] Record at 1080p
- [ ] No loading spinners (edit out or speed up)
- [ ] Smooth cursor movements
- [ ] Test the flow 3x before recording

## Common Mistakes to Avoid

### ‚ùå Don't Do This:

1. **Just screen recording** (no face = no connection)
2. **Talking too fast** (nervous energy, hard to follow)
3. **Showing everything** (pick ONE core flow)
4. **No clear CTA** (viewers don't know what to do next)
5. **Bad audio** (echo, background noise, too quiet)
6. **Over 90 seconds** (viewers drop off at 1:30)
7. **No metrics** (just features, no value proof)

### ‚úÖ Do This Instead:

1. **Start with your face** (builds trust immediately)
2. **Speak slowly** (rehearse 3x, time yourself)
3. **Focus on magic moment** (one wow moment is enough)
4. **End with face + URL** (clear next step)
5. **Test audio first** (record 10-sec test clip)
6. **Edit ruthlessly** (keep it tight, 88-90 seconds)
7. **Show the impact** (saved X hours, made $Y, rated Z stars)

**DO:**

- ‚ùå Show loading spinners for 30 seconds (boring)

- ‚ùå Go silent while app loads (keep talking)

## Backup Plans

### What If Internet Fails?

**Plan A:** Use mobile hotspot
- [ ] Test hotspot speed before demo day
- [ ] Have phone charged and ready

**Plan B:** Pre-recorded demo video
- [ ] Record full 6-minute demo
- [ ] Have video on laptop + USB drive
- [ ] Narrate over the video live

### What If Demo Breaks?

**Plan A:** Have test account with pre-loaded data
- [ ] Create "demo" user account
- [ ] Pre-load example data
- [ ] Test this account 5x before presentation

**Plan B:** Screenshots/video
- [ ] Take screenshots of each step
- [ ] Walk through screenshots if live demo fails
- [ ] Explain: "Here's what it looks like when it works"

### What If Time Runs Over?

**Cut the case study:**
- Show only Slide 1 (Architecture)
- Say: "Full details in our GitHub README"
- End with CTA and URL

## Team Coordination

### Speaking Roles (Recommended)

**10-minute presentations work best with role assignments:**

| Section         | Speaker      | Backup       |
|----------------|--------------|--------------|
| Introduction    | Person 1     | Person 2     |
| Video (silent)  | All (watch)  | All (watch)  |
| Live Demo       | Person 2     | Person 1     |
| Case Study      | Person 3     | Person 1     |

**Why 3 speakers?**
- Shows team collaboration
- Keeps audience engaged (different voices)
- Reduces pressure on any one person

**Meeting Before Presentation:**
- [ ] Decide who speaks when
- [ ] Practice handoffs between speakers
- [ ] Agree on backup plan if someone freezes
- [ ] Test all technology together
